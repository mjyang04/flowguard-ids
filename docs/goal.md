# FlowGuard IDS — 毕业设计论文指南

## 论文标题

**基于 CNN-BiLSTM-SE 的轻量级网络入侵检测系统设计与实现——面向跨数据集泛化**

英文：Design and Implementation of a Lightweight CNN-BiLSTM-SE Network Intrusion Detection System with Cross-Dataset Generalization

## 核心贡献

1. **统一特征空间**：设计 55 维 NetFlow 特征对齐方案，使 CICIDS2017 和 UNSW-NB15 可在同一模型下训练与测试
2. **跨数据集泛化评估**：不仅评估同数据集性能，更关注模型在不同网络环境下的检测能力迁移
3. **不平衡感知评估体系**：提出以 `recall@FAR≤1%` 为核心的评估指标，替代传统准确率，更符合 IDS 实际需求
4. **轻量级深度模型**：CNN-BiLSTM-SE 仅 0.72M 参数，兼顾检测性能与部署可行性

## 模型优化目标

### 主要优化指标（按优先级）

| 优先级 | 指标 | 目标 | 说明 |
|--------|------|------|------|
| 1 | `recall_at_far_1pct`（跨数据集） | 尽可能高 | **论文核心**，模型在新环境下的检测能力 |
| 2 | `recall_at_far_1pct`（同数据集） | ≥ 0.98 | 基线性能，不能太差 |
| 3 | `pr_auc`（跨数据集） | 尽可能高 | 全局排序质量 |
| 4 | `avg_attack_recall`（跨数据集） | 尽可能高 | 攻击类别的平均召回率 |

### 约束条件

- 参数量保持轻量（< 5M）
- VRAM 占用合理（< 4GB，适配 RTX 3060 6GB）
- 同数据集性能不能显著低于文献报告的水平

### Autoresearch 调参方向

当前通过 autoresearch 框架自动搜索超参数，重点方向：

- LSTM 隐藏层大小（64 / 128 / 256）
- 卷积通道数和层数
- 学习率和优化器（Adam / AdamW）
- 调度器（Plateau / Cosine）
- Dropout 比率
- 类别权重策略

**关键原则**：每次实验同时记录同数据集和跨数据集两组指标。跨数据集指标是决策依据，同数据集指标是安全底线。

## 写作注意事项

1. **跨数据集泛化是主线**，所有实验设计和分析都要围绕这个展开
2. **不要过度宣称创新**，诚实描述工作：统一特征空间是工程贡献，SE 是已有技术的应用，评估指标是针对 IDS 场景的合理选择
3. **实验结果要诚实**，如果跨数据集性能下降明显，分析原因比掩盖更有价值
4. **与文献的对比要公平**，注明实验设定的差异（如训练集/测试集划分、是否跨数据集）
5. **SHAP 部分点到即止**，不是论文重点，一节即可
